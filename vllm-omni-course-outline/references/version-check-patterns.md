# vLLM-Omni 版本核对模板

这个参考文件用于把“先核对版本再写课纲”落到实际输出，而不是停留在口头提醒。

## 核对顺序

涉及“最新版本”“当前版本”“指定版本”“main 分支”“最近代码”时，默认按下面顺序核对：

1. `官方 release/tag`
2. `README Latest News`
3. `本地 tag/commit`

如果三者信息不完全一致，优先以官方 release/tag 为准，并在课程说明里写明是否同时参考了本地当前代码。

## 需要落到最终输出的字段

版本化课纲默认至少写出：

- `版本号`
- `官方发布日期`
- `是否同时参考本地仓库当前代码`
- `该版本值得强化的课程主题`

## 默认输出模板

如果用户要求结合版本，开头默认写成：

- `基于 vLLM-Omni vX.Y.Z`
- `官方发布日期：YYYY-MM-DD`
- `本次课纲额外强化：主题 A、主题 B、主题 C`

如果同时参考了本地仓库代码，再补一句：

- `同时参考本地仓库当前代码：<tag 或 commit>`

## 推荐写法

推荐把版本信息写在课程说明或导论里，不要散落在各课标题中。

推荐写法示例：

`基于 vLLM-Omni v0.18.0`

`官方发布日期：2026-03-XX`

`同时参考本地仓库当前代码：main@<short_commit>`

`本次课纲额外强化：entrypoint 重构、scheduler/runtime 清理、统一 quantization、Diffusion 执行链路、生产化能力`

## 常见错误

- 只写“基于最新版本”，不写实际版本号
- 只写版本号，不写发布日期
- 只看 README，不核对官方 release/tag
- 引用了本地 main 分支代码，却没有说明“同时参考当前代码”
- 没写该版本需要强化的课程主题，导致版本核对对课纲没有实际帮助
